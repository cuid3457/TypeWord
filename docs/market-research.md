# TypeWord - 시장조사/수요조사 요청서

아래 내용을 바탕으로 이 앱의 시장성, 수요, 경쟁 환경, 성장 가능성을 분석해 주세요.
냉정하고 객관적인 시각으로 평가해 주세요. 긍정적인 면뿐 아니라 리스크와 약점도 지적해 주세요.

---

## 1. 앱 개요

- **이름**: TypeWord
- **한 줄 설명**: 원서(외국어 책) 읽는 사람을 위한 AI 단어장 + 간격 반복(SRS) 복습 앱
- **플랫폼**: iOS / Android (React Native, Expo)
- **지원 언어**: 14개 (영어, 한국어, 일본어, 중국어, 스페인어, 프랑스어, 독일어, 이탈리아어, 포르투갈어, 러시아어, 베트남어, 인도네시아어, 태국어, 아랍어)
- **출시 상태**: 개발 완료, 스토어 출시 준비 중

---

## 2. 핵심 기능

### 2-1. 단어장 관리
- 책 단위로 단어장 생성 (제목, 원서 언어, 번역 언어 설정)
- 단어 검색 시 AI(OpenAI)가 일반적 빈도/중요도 기반으로 뜻을 우선순위 정렬
  - 문장을 함께 입력하면 해당 문맥에 맞는 뜻을 우선 제공
- 단어당 최대 5개 의미, IPA 발음 기호, 예문, 유의어, 반의어 제공
- TTS(Text-to-Speech) 발음 재생 (디바이스 내장, API 비용 없음)
- 단어장 내 단어 검색/필터링

### 2-2. AI 단어 검색 구조
- **즉시 저장**: 검색 즉시 기본 뜻을 저장하고, 예문/유의어/반의어는 백그라운드에서 보강
- **SSE 스트리밍**: 검색 결과가 실시간으로 화면에 표시 (로딩 대기 최소화)
- **캐시**: 동일 단어+언어 조합은 서버 캐시 → 재검색 시 즉시 응답 (장르 구분 없이 통합 캐시로 히트율 극대화)
- **폴백**: AI 실패 시 무료 사전 API(Free Dictionary)로 대체

### 2-3. SRS 복습 (간격 반복 시스템)
- SM-2 알고리즘 기반 자동 복습 스케줄링
- 플래시카드 UI: 단어 → 탭하여 뜻 확인 → "알겠어요" / "아직 학습 중" 2버튼 평가
- 복습 완료 시 통계 요약 (맞힌 수, 학습 중인 수)
- 복습할 단어 수 탭 뱃지 표시
- 삭제된 단어장의 단어는 복습에서 자동 제외

### 2-4. 기타
- 온보딩: 모국어 선택 → 원서 언어 선택 → 바로 사용 시작 (회원가입 불필요)
- 익명 인증 (Supabase): 앱 설치 즉시 사용 가능, 나중에 계정 연동 가능
- 다크 모드 자동 지원
- 개인정보처리방침 내장
- AdMob 배너 광고 (단어장 목록, 단어 추가 화면 하단)

## 3. 타겟 사용자

### 주요 타겟
- **원서(외국어 책) 읽는 사람**: 영어 원서, 일본어 소설, 프랑스어 문학 등
- **유학생/어학 학습자**: 전공 서적을 외국어로 읽는 대학생
- **시험 준비생**: TOEFL, IELTS, JLPT 등 어휘력이 중요한 시험 준비

### 사용 시나리오
1. 원서를 읽다가 모르는 단어 발견
2. TypeWord에서 단어 검색 → AI가 정리한 뜻 확인
3. 저장 버튼 탭 → 즉시 단어장에 추가
4. 다음 날 복습 탭에 알림 뱃지 → 플래시카드로 복습
5. 반복하며 어휘력 축적

## 5. 차별화 포인트

1. **AI 기반 정의**: 일반 사전과 달리 AI가 학습자 눈높이에 맞춘 간결한 정의 + 문장 입력 시 문맥 반영
2. **검색 → 저장 → 복습 원스톱**: 사전 앱 + 단어장 앱 + 복습 앱을 하나로 통합
3. **제로 설정**: 앱 설치 → 모국어 선택 → 바로 사용. 회원가입/이메일 불필요
4. **원서 독자 특화**: 범용 사전이 아닌, "책 읽는 사람"에 최적화된 UX

## 7. 분석 요청 항목

아래 항목들에 대해 분석해 주세요:

### 시장 규모
- 전 세계 외국어 학습 앱 시장 규모와 성장률
- 원서 독자 시장 규모 (영어 원서, 일본어 원서 등)
- 어휘 학습 앱 세그먼트의 TAM/SAM/SOM

### 수요 검증
- 이 앱이 해결하는 문제(pain point)가 실제로 존재하는가?
- 기존 솔루션(Anki, 사전 앱 등) 대비 전환 동기가 충분한가?
- 14개 언어 지원이 글로벌 확장에 유리한가?

### 경쟁 환경
- 직접 경쟁자 (유사한 포지셔닝의 앱)
- 간접 경쟁자 (Anki, Quizlet, 사전 앱, Kindle 사전 등)
- 진입 장벽과 모방 가능성

### 리스크
- 사용자 획득 비용(CAC) 대비 수익성
- AI API 비용 스케일링 리스크
- 사용자 리텐션 (단어장 앱의 일반적인 이탈률)
- 레드오션 vs 블루오션 판단

### 성장 전략 제안
- 초기 사용자 획득 채널 (ASO, SNS, 커뮤니티 등)
- 바이럴 요소가 있는가?
- 단계별 성장 로드맵 제안

### 총평
- 이 앱을 출시했을 때 성공 가능성 (1~10점)
- 가장 큰 강점과 가장 큰 약점
- "이것만 바꾸면 성공 확률이 올라간다"는 핵심 제안 1~3가지

## 8. 기술 스택 참고
